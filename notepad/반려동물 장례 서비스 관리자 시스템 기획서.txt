# 반려동물 장례 서비스 관리자 시스템 기획서

## 1. 프로젝트 개요

### 1.1 목적
- 반려동물 장례 서비스의 효율적인 관리와 운영
- 예약부터 정산까지 일원화된 관리 시스템 구축
- 데이터 기반 의사결정 지원

### 1.2 개발 환경
- Backend: Django 4.2
- Database: MySQL 8.0
- Frontend: Tailwind CSS
- Deployment: 미정

### 1.3 타겟 사용자
- 서비스 관리자
- 장례식장 직원
- 고객 응대 담당자

## 2. 시스템 구조

### 2.1 사용자 인증 시스템
- JWT 기반 인증
- Role 기반 권한 관리
  - 슈퍼관리자: 모든 권한
  - 일반관리자: 예약/서비스 관리
  - 직원: 예약 확인/처리

### 2.2 데이터베이스 설계
- Users
  - id, username, password, role, created_at
- Pets
  - id, name, species, weight, death_date, death_cause
- Services
  - id, name, description, base_price, status
- OptionGroups
  - id, service_id, name, description
- Options
  - id, group_id, name, price, is_default
- Reservations
  - id, pet_id, service_id, status, total_price, signature
- Notifications
  - id, reservation_id, type, content, sent_at

## 3. 핵심 기능 명세

### 3.1 예약 관리
#### 예약 프로세스
1. 보호자 정보 입력
   - 필수: 이름, 연락처, 이메일
   - 선택: 주소(카카오 주소 API 연동)

2. 반려동물 정보 입력
   - 기본정보: 이름, 품종, 나이, 성별, 몸무게
   - 사망관련: 사망일, 사망경위
   - 추가정보: 방문경로, 경유업체, 장례확인서(발급/미발급)

3. 예식 선택
   - 4가지 예식 패키지 제공
   - 무게별 추가 비용 자동 계산
   - 실시간 총액 계산

4. 최종 확인
   - 디지털 서명(react-signature-canvas)
   - PDF 출력 기능

### 3.2 서비스 관리
#### 예식 패키지 관리
- 기본/고급/최고급/호화 예식 구성
- 옵션 그룹별 기본 옵션 설정
- 가격 정책 설정

#### 옵션 그룹 관리
- 생화/관/유골함/이불/액자 세트
- 그룹별 옵션 CRUD
- 기본 옵션 지정

### 3.3 통계 및 리포트
#### 대시보드
- 일/주/월간 매출 추이
- 서비스별 이용 현황
- 고객 유입 경로 분석

#### 데이터 추출
- 기간별 예약 현황 CSV export
- 매출 리포트 PDF 생성

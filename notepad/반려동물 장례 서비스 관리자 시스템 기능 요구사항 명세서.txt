# 반려동물 장례 서비스 관리자 시스템 기능 요구사항 명세서

## 1. 사용자 인증 및 권한 관리

### 1.1 로그인
- REQ-1.1.1: 아이디/비밀번호 로그인 기능
- REQ-1.1.2: 카카오 소셜 로그인 연동
- REQ-1.1.3: 구글 소셜 로그인 연동
- REQ-1.1.4: 자동 로그인 기능 (쿠키/로컬 스토리지 활용)
- REQ-1.1.5: 로그인 실패 시 오류 메시지 표시

### 1.2 회원가입
- REQ-1.2.1: 기본 정보 입력 (이름, 아이디, 비밀번호)
- REQ-1.2.2: 비밀번호 강도 실시간 검증 (대문자, 특수문자, 숫자 포함)
- REQ-1.2.3: 소셜 계정 연동 회원가입
- REQ-1.2.4: 이메일 인증 시스템

### 1.3 권한 관리
- REQ-1.3.1: 역할 기반 접근 제어 (RBAC)
- REQ-1.3.2: 슈퍼관리자 권한 관리
- REQ-1.3.3: 일반관리자 권한 관리
- REQ-1.3.4: 직원 권한 관리

## 2. 예약 관리

### 2.1 예약 생성
- REQ-2.1.1: 4단계 예약 프로세스 구현
- REQ-2.1.2: 보호자 정보 입력 및 검증
  - 필수: 이름, 연락처, 이메일
  - 선택: 주소 (카카오 주소 API)
- REQ-2.1.3: 반려동물 정보 입력 및 검증
  - 기본: 이름, 품종, 나이, 성별, 몸무게
  - 사망: 사망일, 사망경위(자연사/병사/사고사/안락사)
  - 추가: 방문경로, 경유업체, 장례확인서
- REQ-2.1.4: 예식 선택 시스템
  - 4가지 예식 패키지 표시
  - 무게별 추가 비용 자동 계산
  - 실시간 금액 계산
- REQ-2.1.5: 디지털 서명 시스템

### 2.2 예약 관리
- REQ-2.2.1: 예약 목록 필터링 (날짜별, 상태별)
- REQ-2.2.2: 예약 상태 관리 (대기/진행/완료)
- REQ-2.2.3: CSV 데이터 추출
